Threads

  • Thread is an execution unit that is part of a process

    • A process cna have multiple threads, which can execute at the same time

  • Thread is an execution units managed by OS scheduler

  • Thread is a “lightweight process”

    • Threads in a prcess share the same address space and many resources
      • While different processes do not share

  • Typical OS design

    • Multiple processes
    • Each process contains multiple threads

Address Space

  • Logical address – generated by the CPU; also referred to as virtual address
  • Physical address –address seen by the memory unit

Comparing Process and Thread

PROCESS THREAD
Processes are heavyweight operations Threads are lighter weight operation
Each process has its own memory space Thread use the memory of the process they belong to
Inter -Process communication is slow as processes have different memory addresses Inter-thread communication can be faster than inter-process communication because threads of the same process share memory with the porcess they belong to
Context switching between processes is more expensive Context switching between threads of the same process is less expensive
Processes dont share memory with ohter processes Threads share memory with other threads of the same process

Why Multi-threads in a Process

For parallel execution

  • Some Program has inherent paralleism
    • e.g. adding two large arrays, or incrementing the value of each element in the array by some amount
  • If executed on computer with multiple CPU cores, we can run each thread on a CPU core, to speedup the the execution.

For higher CPU utilization in the presence of I/O blocking

  • I/O operation is usually slow, CPU wasted when waiting for I/O

  • Instead of waiting, a program may be able to do some thing else

    • utilizing the CPU to perform computation
    • issuing other I/O requests

  • Using multiple threads to avoid getting stuck: while one thread waits, the CPU scheduler can switch to other threads to do something useful

  • Helpful even if there is only one CPU core

Parallelism vs Concurrency

  • Parallelism: running multiple threads/processes in parallel over different CPU cores
  • Concurrency: running multiple threads/processes at the same time, even on single CPU core, by interleaving their executions

User Thread vs Kernel Threads

User Thread

  • Thread management done by user-level threads library
  • example:
    • POSIX Pthreads
    • Java threads

Kernel Threads

  • Managed by the Kernel

  • User threads are mapped to kernel thrads

User Thread vs Kernel Treads

User Thread Kernel Thread
User thread are implemented by users Kernel threads are implemented inside OS
Oriented to users;users program with user threads, but do not worry about hte implementation details Oriented to kernel; OS kernel actually manages the threads
OS doesnt recognize user level threads Kernel threads are recognized by OS
Possible to chedule the user threads in user level wothout entering the kernel mode;in this case, the scheduling functionality is limited The OS kernel has full functionality to manage kernel threads
If one user level thread perform blocking operation then entire process will be blocked If one kernel thread perform blocking operation then another thread can continue execution.
Context switch overhead is lower Context switch overhead is higher

Multithreading Models

One-to-One:

-Each user-level thread mapped to a kernel thread

  • Different user threads can execute in parallel simultaneously
  • Expensive in resource: Creating a user thread requires a kernel thread
  • Examples: Windows NT/XP/2000, Linux, Solaris 9 and later

Many-to-One:

  • Many user threads mapped to one kernel thread
    • One thread can access kernel at a time
    • Multiple thread can not run in parallel
    • Examples: Solaris Green Treads, GNU Portable Threads

Many-to-Many:

  • Allows many user level threads to be mapped to many kernel threads
    • Allows OS to create a sufficient number of kernel threads
    • Examples: Solaris prior to version 9, Windows NT/2000 with the ThreadFiber package

Combined Model:

  • Similar to M:M, except that it allows a user thread to be bound to kernel thread
    • Coexistence of one-one nad many-many
    • Examples: IRIX, HP-UX, Tru64 UNIX, Solaris 8 and earlier

Semantics of fork() and exec()

  • Does fork() duplicate only the calling thread or all threads?
    • Some OS has tow versions of fork()
      • one duplicates all threads
      • one only duplicate the invoking thread
  • exec() works the same as process
    • The porgram specified in parameter will replace the entire process
      • If all thrads duplicated, then replace for all threads

Thread Cancellation

  • Terminating a thread before it has finished

    • Example: in Pthread, use pthread_cancel() to cancel another thread

  • Two general approaches

    • Asynchronous cancellation terminates the target thread at any time
      • supposedly terminate immediately, but not guaranteed
    • Deferred cancellation allows the target thread to be cancelled at certain cancellation point
      • To avoid improper cancellation e.g. holding unreleased resource

Thread Pools

  • Create a number of threads in a pool where they await to do tasks
  • Advantages:
    • Usually faster than creating a new thread
    • Allows to bound the number of therads in the application
  • Example: Web servers, Oracle DB, OpenMP

Pthreads

  • POSIX Thread(Pthread)

    • A POSIX standard API for thread creation and synchronization

  • API specifies behavior of the thread library, implementation is up to development of the library

  • Common in UNIX OS

    • e.g. Solaris, Linux, Mac OS X
    • these OS provides the Ptherad library to user

  • Thread Creation

  • Thread Completion

Therad Join

  • “joining” is one way to accomplish synchronization between threads.
  • The pthread_join() subroutine blocks the calling thread until the tread identified by thread identifier terminates
  • The worker tread’s return value can be obtained by retval if it was specified in the worker’s call to pthread_exit().
  • It is a logical error to attempt simltanceous multiple joins one the same worker thread.

Contex Switch

  • Context of a thread

    • CPU register
    • Stack

  • Context of a process

    • CPU registers
    • Entire address space(code, data, heap, stack)
    • Many other resource: kernel resource, I/O, file

Context Protection for Interrupts

  • When an interrupt occurs, before switching to ISR, registers are saved to the stack

    • hardware assisted
    • Include:PC(program counter) SP(stack pointer)

  • Why?

    • Need to protect the registers: ISR needs to use these register
    • Interrupt is asynchronous, so the compiler does not know which registers to be protected
      • Different from the function call

System Tick(SysTick)

  • time unit that OS timers and delays are base on

  • work to be done(when switching from task 1 to task 2)
    • save the registers of the task 1 to stack 1
    • restoring the registers of task 2 from stack 2
    • using the PC value stored stack 2 to resume task 2
      • resume from the point where this task was interrupted last time

Java Thread

  • Java threads are managed by the JVM

    • JVM: Java Virtual Machine

  • Java threads may be created by:

    • Extending Thread class
    • Implementing the Runnable interface

  • Implemented by many-to-one or many-to-many model
