進程（Process）

進程和線程（Process and Thread)

  • 進程
    • 一個執行的程序（a program in execution)
  • 線程
    • 線程是進程的執行單位(A thread is the unit of execution whithin a Process)

進程（Process)

  • 進程包括

    • 程序計數器
    • 棧
    • 數據
    • …

進程狀態（Process State)

  • 作為一個進程執行，它會改變狀態
    • 新（new):進程開始被創建
    • 準備（ready): 進程在等待CPU分配任務
    • 運行（running): 指令正在被執行
    • 等候（waiting): 進程等待某些時間發生
    • 終結 (terminated): 進程已結束執行

進程控制塊 （Process Control Block) PCB

  • 每個進程都被展示在OS里的進程控制塊

    • 或者成為任務控制塊

  • 每個進程在OS的信息紀錄如下：

    • 進程ID(Process ID (PID))
    • 進程狀態（Process state)
    • 程序計數器 (program counter)
    • 寄存器數值 (Register values)
    • CPU調度信息 (CPU scheduling information)(e.g. priority)
    • 打開的文件列表 (List of open files)
    • I/O 狀態信息 (I/O status information)

調度（Scheduling)

進程調度 (Process Scheduling)

  • 一些隊列涉及進程調度
    • 工作列隊 (Job queue) - 系統中所有進程的集合
    • 預備列隊 (Ready queue) - 主要內存進程的集合，預備和等候去執行
    • 設備列隊 (Device queue) - 等候I/O設備進程的集合

  • 進程遷移在這些列隊中

調度程序 (Scheduler)

  • 長期調度程序（long-term scheduler） (or job scheduler) - 選擇進程應該進入預備列隊

    • 很少調用
    • 安排執行在某些工作在某些時間
    • 控制多程序的程度

  • 短期調度程序 (short-term scheduler) (or CPU scheduler) - 選擇進程應該被執行和分配CPU

    • 經常調用
    • 創建並發（concurrency), 切換必須足夠快

CPU綁定和I/O綁定

  • 進程可以分為
    • CPU綁定進程 (CPU-bound process): 花費更多時間在計算
      • 很少有長時間CPU衝突
    • I/O綁定進程 (I/O-bound process): 花費更多時間在I/O多於計算
      • 有許多短時間的CPU衝突

內容切換 (Context Switch)

  • 當CPU切換去其他進程， 系統會保存舊進程的狀態和載入已保存狀態的新進程

    • 內容切換時間是開銷， 切換時沒有幹活
    • 時間依賴於硬件支持

  • 內容切換設想例子

  • 父進程創建子進程

進程創建 (Process Creation)

  • 子進程可能會複製父進程的內存空間，或者載入新的內存內容，

  • UNIX 例子

    • fork 系統稱為創建新進程
    • exec 系統稱為使用在fork去取代進程內存空間在新程序和執行

  • 步驟：

    1. 載入一個程序代碼進入內存：

      • 程序最初可執行格式留在磁盤上
      • OS 平台延遲加載進程
        • 加載代碼塊或者數據只有他們在程序執行需要的時候

    2. 程序運行棧被分配

      • 使用棧為本地變量(local varoables)，函數參數(fucntion parameters)和返回地址(return address)
      • 初始化棧用參數 argc 和argv array 在main()函數

    3. 程序的堆被創建

      • 為了明確請求動態分配數據
      • 程序請求空間調用 malloc()函數和釋放他們調用free()函數

    4. OS做一些其他的初始化任務

      • I/O 設置
        • 每一個進程預設有3個打開文件描述
        • 標準輸入，輸出和錯誤

    5. 開始程序運行入口

      • 稱為 main()
      • OS 將CPU的控制權轉移給新創建的進程

安卓的進程 (Process on Android)

  • 默認情況下，每個應用運行在他們擁有的Linux 進程

  • 有一些應用可能會使用幾個進程

進程終結 (Process Termination)

  • 進程執行最後狀態和詢問系統刪除它（exit)

    • 輸出數據從子進程到父進程 （wait)
    • 進程資源被OS取消進程（de-allocated)

  • 父進程可以終止子進程的執行（abort）

    • 發生於
      • 子進程超過分配資源
      • 任務分配給子進程不再需要
      • 父進程存在
        • 有一些OS 不允許子進程繼續運行當父進程被終結時

通訊（Communication)

進程合作（Processes Cooperation)

  • 獨立進程不能影響或者被影響於其他進程

  • 合作進程可以影響或者被影響於其他進程

  • 進程合作的好處

    • 信息共享
    • 計算速度上升
    • 模塊化
    • 方便

生產者-消費者問題（Producer-Consumer Problem)

  • 生產者生產信息被消費者消費
    • 無界緩衝區沒有實際限制緩衝區大小
    • 有界緩衝區假設了一個固定大小的緩衝區

進程間通訊 (Interprocess Communication)(IPC)

  • 一個機制為了進程通訊和同步
  • 信息系統 - 進程間通訊不需要求助於共享變量
  • IPC 設施提供兩個操作
    • 發送（send)
    • 接收（receive)
  • P 和 Q 希望通訊，他們需要
    • 建立一個通訊連接
    • 交換信息通過發送和接收

實現問題 (implementation)

  • 怎樣建立連接？
  • 連接可以相關聯多過兩個進程？
  • 多少個連接可以在每一對進程通訊？
  • 連接的容量是什麼？
  • 信息大小在連接可以容納固定或者變量？

通訊模型（Communication Models)

信息傳遞模型 分享內存模型
信息傳遞設施是用於通訊 分享內存區域用於通訊
它能用於分佈式環境，通訊進程寄留在遠端機器通過網絡連接 它用於通訊在進程間在一個處理器或者多個處理器系統，通訊進程在同一個機器
OS提供通信和同步機制執行在通訊進程間 需要程序員去寫程序明確調度數據訪問在不同的進程
它是耗時的，信息傳遞實現通過核心干涉 它提供了最大速度計算於通訊，通過共享內存和系統調用只需要建立共享內存

直接通訊 (Direct Communication)

  • 進程必須明確互相命名

    • send(P, message) - 發送信息去進程P
    • receive(Q, message) - 接受一個信息來自於進程Q

  • 通訊連接的特性：

    • 自動地建立連接
    • 一個連接相關於具體一對通訊進程
    • 每個對存在具體的連接
    • 連接可能是單向的，但通常是雙向的

間接通訊 (Indirect Communication)

  • 信息直接和接收來自於郵箱（或者成為port)

    • 每個郵箱有獨立的ID
    • 進程能通訊只有他們共享一個郵箱

  • 通訊連接的特性：

    • 連接只建立在於進程共享一個共同的郵箱
    • 一個連接可能相關很多進程
    • 每個進程對可能分享多個通訊連接
    • 連接可能是單向或雙向

  • 郵箱操作

    • 創建一個新的郵箱
    • 發送和接收信息通過郵箱
    • 摧毀一個郵箱

  • 原語被定為

    • send(A, message) - 發送一個信息去郵箱A
    • receive(A, message) - 接收一個信息來自郵箱A

  • 郵箱共享

    • P1, P2, P3 共享郵箱A
    • P1, 發送， P2和P3接收
    • P2和P3接收到信息

  • 可能解決方案

    • 允許一個連接相關於最多兩個進程
    • 允許只有一個進程在一個時間執行接收操作
    • 允許系統選擇隨意接收者，發送者提示誰為接收者

同步 (Synchronization)

  • 信息傳遞要麼是阻塞或者不阻塞

    • 阻塞被考慮在同步
      • 阻塞發送是發送者阻塞，直到信息被接收
      • 阻塞接收是接受者阻塞，直到信息可用

  • 非阻塞考慮在異步

    • 非阻塞發送是發送者發送信息然後繼續
    • 非阻塞接收是接收者接收一個有效信息或者null

客戶-服務器通訊 （Client-Server Communication)

  • 互聯網接套

    • 一個接套被定義一個終端為通訊
    • IP地址串聯和port

  • Unix接套

    • 使用必要地相同功能調用像互聯網接套
    • 所有的通訊發生完全在OS的核心，而不是使用一個潛在的網絡協議
    • 雙向FIFO隊列

遠程過程調用 (Remote Procedure Calls)

  • RPC 抽象過程調用在網絡系統進程中
  • 客戶端存根定位服務器和整理參數
  • 服務器端存根接收這個信息，打開整理參數和運行步驟在服務器
